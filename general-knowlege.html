常识大补习

尝试用最简单明了的语言解释常见名词.(作为谷歌营全员学习计划Linux教学的一部分.真的有大二SE学生问出内存是什么.服务器是什么这种问题.)

硬件

拆开台式机或笔记本可以看到以下零件:

主板

把芯片\电路板放到一起的板子

CPU

用来运算的,通常上面有个风扇给它降温.和金属散热片之间一般涂硅胶用于导热.

硬盘

用来储存所有文件的.包括操作系统\应用程序\你的文档\电影\照片\音乐

内存

程序运行时储存数据的部件.跟硬盘的区别是内存读写快上百倍.写C时声明变量如int i; 或是malloc函数.都是向系统要内存来存放数据.

软件

扣扣\迅雷\浏览器都是软件.有时也称作应用程序.应用程序特指给用户用的.区别于看不到的数据库软件\Web Server等软件.

操作系统

用来管理其他程序以及硬件的程序.开机也就是启动操作系统.

虚拟机

像运行QQ一样运行若干个其他操作系统.典型虚拟机软件:Virtualbox

服务器

服务器有时候指在机房躺着的那台机器.[硬件].它除了性能更强外与我们用的电脑没有本质区别.由于功能主要用于提供服务,所以叫做服务器.一般是没有显示器的.通过远程控制软件进行管理.win中一般为自带的"远程桌面",*nix中一般为ssh. 也有时候指在机器上运行的服务性软件.像提供网页给我们看的Web Server.代表性的为Nginx.保存我们各种数据[帐号\密码\发的说说\游戏的装备\淘宝下的订单\给的评论]的数据库.典型代表为MySQL\MongoDB.

服务器一般不自己买硬件组装.而是更简单快速廉价的租用专业团队提供的虚拟服务器(VPS),断电\散热\网络等等都不用我们操心.只管付钱用就好.由于是虚拟机,可以不改动硬件就能更改它的配置.[增加带宽\内存\CPU核心.]

云计算/云存储/云服务

所谓云,就是指把很多机子连起来做成一个集群.这些机子可以动态的给他们分配任务.国庆7天某游戏APP很火爆,那么就自动的多分配一些机子帮助分担压力.平时闲下来又可以给其他应用使用,提高了效率和灵活程度.总体运算/存储能力还可以通过增减机子来灵活调整.并且,这些改动完全不影响在上面运行的应用.应用不需要改一行代码.典型代表:亚马逊云服务Amazon Web Services (AWS)\谷歌APP Egine\阿里云. 云存储也是一样的.把诸多文件分散到一大批机子里(集群).增强稳定性和可用率. 云服务就是把这些技术出租给开发者使用的服务.

大数据

顾名思义,就是有大量的数据,从而可以统计上发现一些规律. 经典的例子是有了每个顾客的购物数据后发现啤酒和尿布经常被一起买.然后超市据此把啤酒和尿布放到一起进一步提高销量. 更诡异的例子是发现下雨的时候面包卖的好,晴天三明治买的好.其中的因果难以考证,但,也没必要考证,知道晴天做多少三明治,雨天做多少面包马上就能提高利润. 还有个例子是谷歌通过人们的搜索请求以及人们的地理位置信息发现流感的发源地.原因是人们感冒了会去谷歌上搜相应的症状\办法. 大数据大概就是大量样本消除随机因素.加多维度数据找出关联. 还有利用Twitter的推文来预测奥巴马是否当选的.SNS数据预测苏格兰公投结果的.各种数据(包括人们下的赌注\球队\球员历史数据)综合预测世界杯结果的.

人工智能

一个/类算法是否称得上人工智能没有绝对标准.我们人认为它做到了人类才能做到的事就说它是人工智能.图灵测试就是基于这个定义.图灵测试大意就是当一个AI成功的让人类误以为他是人类,那么就说这个AI达到了人类的智能.[很想在这里说个题外话,有句话说"我知道我自己是有思维的,但其他人我就不能确定了.",你们感受下.].

目前比较强的AI有无人驾驶汽车.12年Andrew ng团队的神经网络在看了一千万张youtube视频中的图片后,有了猫的概念.并能分辨2000多种物品.几乎将正确率提高了一倍.这里提供点关键字供你们搜索.[神经网络, 深度学习]. 如果说基于概率统计等数学方法的算法仍然像算法的话.神经网络则更像生物的大脑.对于人工智能感兴趣的同学可以看看<How to Create a Mind>

获取知识的途径

这个应该说过无数遍了.有问题\有不懂的概念\名词.先搜一下.看看大概是个什么玩意.如果搜出来的解释中又有很多不懂的概念\名词.再继续搜这些名词.直到你满足为止. 只要你大的概念有了.以后再有新的名词出现就可以很快归类到某个大概念之下了.而概念有了,碰到问题也不至于束手无策.至少知道大概需要哪些信息/知识.这个过程比较困难.但对于有兴趣有好奇心的人来说是个充满乐趣的过程.[你找别人帮忙解决问题,比如装个机什么的,他可能也没装过,但他如果有启动盘\系统\分区的概念的话,搜个教程依样画葫芦是很简单的一件事.而你找他帮忙自己什么都不用管.就错过了一次知道这些概念\学会装机的机会.更重要的是概念的缺失.] 举个例子.编译自己写的程序时编译器报错,[很多人这个时候就不知道怎么办了,找人帮忙就说一句出错了.这种时候我都有想打人的冲动.],这特么又没几个单词.就算一个个翻译搞清楚怎么回事也要不了十分钟.如果函数\编译的概念没有还能顺便知道这些概念.见到报错只知道关掉再试几次的那是用户.作为开发人员你需要把问题都搞清楚.

hello.c: In function ‘main’:
hello.c:15:2: error: ‘for’ loop initial declarations are only allowed in C99 mode
  for (int i = 0; i < count; ++i)
  ^
hello.c:15:2: note: use option -std=c99 or -std=gnu99 to compile your code

这个错误是说当前编译的规范是不支持在for循环内声明变量的.需要添加-std=c99选项改为C99规范. 而一个学弟给我截图的编译错误则更是让人迷醉.

1.英文很重要.多看多查就会了.

2.对于普通用户.要竭力减少他犯错的机会.比如这时候提示他保存文件.
